This page documents every type and interface exported from `lib/types.ts`, plus the shape-settings utility functions exported from `lib/shape-settings.ts`. These types and utilities are shared across the model builder, work document utilities, storage layer, and the editor UI.

## Shape-settings utilities

These functions are exported from `lib/shape-settings.ts` and are used internally by `buildMedalGroup` and the editor UI.

### resolveShapeSettings()

```typescript
function resolveShapeSettings(
  settings: MedalSettings,
  pathIndex: number,
  summary?: SvgPathSummary,
): ShapeSettings
```

Merges the application defaults with any per-shape patch stored in `settings.shapeSettings` for the given `pathIndex`. The resolved color is derived from the SVG path summary using `getSvgColor` unless a `color` override is present in the patch.

<ParamField path="settings" type="MedalSettings" required>
  The full medal settings object. The function reads `settings.shapeSettings[String(pathIndex)]` to find any stored patch for this path.
</ParamField>

<ParamField path="pathIndex" type="number" required>
  Zero-based index of the SVG path to resolve settings for.
</ParamField>

<ParamField path="summary" type="SvgPathSummary" default="undefined">
  Optional parsed summary of the SVG element. When provided, the resolved fill or stroke color from the SVG is used as the base color before any patch override is applied.
</ParamField>

**Returns** `ShapeSettings` â€” a fully resolved, non-partial settings object. Every field is guaranteed to be present.

The merge order is: `DEFAULT_SHAPE_SETTINGS` â†’ `{ color: getSvgColor(summary) }` â†’ `patch`.

### getSvgColor()

```typescript
function getSvgColor(summary: SvgPathSummary | undefined): string
```

Extracts the best usable color from an SVG path summary. Returns the element's `fill` if it is a usable color (not `"none"`, not a `url(...)` reference), otherwise falls back to the `stroke`, and finally to the default gold color `#d8a737`.

<ParamField path="summary" type="SvgPathSummary | undefined" required>
  The SVG path summary to read colors from. Passing `undefined` returns the default fallback color.
</ParamField>

**Returns** `string` â€” a CSS color value.

### getShapeLabel()

```typescript
function getShapeLabel(summary: SvgPathSummary | undefined): string
```

Returns the display label for a shape. Uses `summary.name` if it is a non-empty string; otherwise falls back to `"Shape N"` where N is `summary.pathIndex + 1`.

<ParamField path="summary" type="SvgPathSummary | undefined" required>
  The SVG path summary to derive a label from.
</ParamField>

**Returns** `string` â€” a human-readable label such as `"Badge outline"` or `"Shape 3"`.

## Shape settings

<AccordionGroup>
  <Accordion title="ShapeSvgAdjustments">

```typescript
export interface ShapeSvgAdjustments {
  circleRadius?: number;
  rectWidth?: number;
  rectHeight?: number;
  rectCornerRadius?: number;
}
```

Optional overrides that rewrite SVG element attributes before the SVG is parsed and extruded. All fields are optional â€” omitting a field leaves the original SVG attribute unchanged.

| Field | Type | Description |
|---|---|---|
| `circleRadius` | `number \| undefined` | Override `r` on `<circle>` elements |
| `rectWidth` | `number \| undefined` | Override `width` on `<rect>` elements, keeping the shape center-anchored |
| `rectHeight` | `number \| undefined` | Override `height` on `<rect>` elements, keeping the shape center-anchored |
| `rectCornerRadius` | `number \| undefined` | Override `rx` and `ry` on `<rect>` elements |

  </Accordion>

  <Accordion title="ShapeSettings">

```typescript
export interface ShapeSettings {
  name?: string;
  thickness: number;
  bevel: number;
  material: MaterialId;
  color: string;
  curveSegments: number;
  bevelSegments: number;
  depthSteps: number;
  zOffset: number;
  visible: boolean;
  deleted: boolean;
  windingMode: ShapeWindingMode;
  svgAdjustments: ShapeSvgAdjustments;
}
```

The full set of per-path settings that control how an individual SVG shape is extruded and rendered.

| Field | Type | Description |
|---|---|---|
| `name` | `string \| undefined` | Display name shown in the editor panel |
| `thickness` | `number` | Extrusion depth in scene units |
| `bevel` | `number` | Bevel size applied to extruded edges, in scene units |
| `material` | `MaterialId` | Which PBR material preset to use |
| `color` | `string` | CSS hex color that overrides the preset's default color |
| `curveSegments` | `number` | Number of curve subdivisions used when approximating curved paths |
| `bevelSegments` | `number` | Number of bevel layers along the extruded edge |
| `depthSteps` | `number` | Number of depth subdivisions along the extrusion axis |
| `zOffset` | `number` | Manual Z-axis offset to layer shapes above or below each other |
| `visible` | `boolean` | Whether the shape is rendered |
| `deleted` | `boolean` | When `true`, the shape is excluded from parsing and rendering |
| `windingMode` | `ShapeWindingMode` | How to interpret the path winding direction |
| `svgAdjustments` | `ShapeSvgAdjustments` | Attribute overrides applied to the source SVG before parsing |

  </Accordion>


A partial version of `ShapeSettings`. This is the type stored in `MedalSettings.shapeSettings` â€” a map from path index string keys to patches that are merged on top of the default shape settings at render time.

  </Accordion>
</AccordionGroup>
