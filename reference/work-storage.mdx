---
title: "work-storage: save, load, and list works in IndexedDB"
description: "Reference for saveWorkDocument(), getWorkDocument(), listSavedWorks(), deleteWorkDocument(), and listWorkDocuments() — the Medal Forge browser storage API."
---

The functions in `lib/work-storage.ts` manage work persistence in the browser's IndexedDB. They handle opening the database, creating the object store and indexes on first use, and mapping raw stored records to validated, normalized `WorkDocument` objects.

<Note>
  This module is marked `"use client"` — it requires a browser environment with IndexedDB. It cannot be imported or called in server components, API routes, or any Node.js context.
</Note>

## Database schema

| Property | Value |
|---|---|
| Database name | `medal-forge-workspace` |
| Version | `1` |
| Object store | `works` |
| Key path | `document.id` |

### Indexes

| Index name | Key path | Unique |
|---|---|---|
| `updatedAt` | `document.updatedAt` | No |
| `title` | `document.title` | No |

The database and object store are created automatically on the first call to any storage function in a given browser profile. You do not need to run a setup step.

---

## saveWorkDocument()

```typescript
async function saveWorkDocument(document: WorkDocument): Promise<void>
```

Upserts a work document into IndexedDB using a `put` operation. If a document with the same `document.id` already exists it is overwritten entirely.

<ParamField path="document" type="WorkDocument" required>
  The work document to save. Must be a valid `WorkDocument` — pass the output of `createWorkDocument()` or `normalizeWorkDocument()`.
</ParamField>

**Returns** `Promise<void>`.

---

## deleteWorkDocument()

```typescript
async function deleteWorkDocument(id: string): Promise<void>
```

Deletes the work document with the given ID from IndexedDB. If no document with that ID exists the operation completes silently — it is a no-op, not an error.

<ParamField path="id" type="string" required>
  The `document.id` of the work to delete.
</ParamField>

**Returns** `Promise<void>`.

---

## getWorkDocument()

```typescript
async function getWorkDocument(id: string): Promise<WorkDocument | null>
```

Retrieves a single work document by ID, validates it with `isWorkDocument()`, and normalizes it with `normalizeWorkDocument()` before returning.

<ParamField path="id" type="string" required>
  The `document.id` of the work to retrieve.
</ParamField>

**Returns** `Promise<WorkDocument | null>` — the normalized document, or `null` if no record exists for the given ID or if the stored record fails validation.

---

## listSavedWorks()

```typescript
async function listSavedWorks(): Promise<SavedWorkSummary[]>
```

Retrieves all stored works, discards any that fail validation, normalizes the rest, and maps each to a lightweight `SavedWorkSummary`. Results are sorted by `updatedAt` descending (most recently updated first).

No parameters.

**Returns** `Promise<SavedWorkSummary[]>` — an array of summaries sorted newest-first. Returns an empty array if no valid works are stored.

Use this function to populate list or grid views. Because summaries omit the raw SVG text, they are significantly smaller than full `WorkDocument` objects.

### SavedWorkSummary interface

```typescript
export interface SavedWorkSummary {
  id: string;
  title: string;
  sourceFileName: string;
  createdAt: string;
  updatedAt: string;
  schemaVersion: WorkDocument["schemaVersion"];
  snapshotDataUrl: string | null;
}
```

<ResponseField name="id" type="string" required>
  Unique work document ID.
</ResponseField>

<ResponseField name="title" type="string" required>
  User-visible title of the work.
</ResponseField>

<ResponseField name="sourceFileName" type="string" required>
  Original filename of the imported SVG (e.g. `"badge.svg"`).
</ResponseField>

<ResponseField name="createdAt" type="string" required>
  ISO 8601 timestamp of when the work was first created.
</ResponseField>

<ResponseField name="updatedAt" type="string" required>
  ISO 8601 timestamp of the most recent save.
</ResponseField>

<ResponseField name="schemaVersion" type="WorkDocument[&quot;schemaVersion&quot;]" required>
  Schema version of the stored document. Currently always `1`.
</ResponseField>

<ResponseField name="snapshotDataUrl" type="string | null" required>
  Base64 PNG data URL of the most recent preview snapshot, or `null` if no snapshot has been captured for this work.
</ResponseField>

## listWorkDocuments()

```typescript
async function listWorkDocuments(): Promise<WorkDocument[]>
```

Retrieves all stored works as full `WorkDocument` objects, including the complete SVG source text. Invalid records are filtered out; valid records are normalized before being returned. Results are returned in insertion order (IndexedDB `getAll()` order).

No parameters.

**Returns** `Promise<WorkDocument[]>` — an array of normalized `WorkDocument` objects. Returns an empty array if no valid works are stored.

Use `listWorkDocuments()` when you need access to the full document payload — for example when exporting all works or performing a batch migration. For display purposes prefer `listSavedWorks()`.

---

<Warning>
  All works are stored locally in the browser's IndexedDB. Clearing site data, storage, or cookies in browser settings permanently deletes every stored work. Always export important works as JSON files before clearing browser storage.
</Warning>

<Tip>
  All storage functions are async and return Promises. Always `await` them or chain `.then()` / `.catch()` handlers. Wrap calls in `try/catch` to handle IndexedDB errors gracefully — for example when the user's storage quota is exceeded.
</Tip>
