`MATERIAL_PRESETS` and `getMaterialPreset` are exported from `lib/materials.ts` and provide the PBR material definitions used when rendering shapes. Each preset carries the color, metalness, and roughness values that are passed directly to `THREE.MeshStandardMaterial` when a mesh is created by `buildMedalGroup`.

---

## MaterialPreset

The shape of each entry in `MATERIAL_PRESETS`, defined in `lib/types.ts`:

```typescript
export interface MaterialPreset {
  id: MaterialId;
  name: string;
  color: string;
  metalness: number;
  roughness: number;
  previewSrc: string;
}
```

---

## MATERIAL_PRESETS

An ordered array of the six built-in presets. The first entry (`brushedGold`) is used as the default material throughout the application.

## getMaterialPreset

```typescript
function getMaterialPreset(id: string): MaterialPreset
```

Looks up a preset by its `id` string and returns it. Falls back to the first preset (`brushedGold`) when no match is found.

### Parameters

<ParamField path="id" type="string" required>
  The `MaterialId` string to look up (e.g. `"brushedGold"`, `"copper"`). Any string is accepted; an unrecognised value returns the `brushedGold` fallback rather than throwing.
</ParamField>

### Return value

The matching `MaterialPreset`, or `MATERIAL_PRESETS[0]` (`brushedGold`) if `id` does not match any entry.

### Source

```typescript
export function getMaterialPreset(id: string): MaterialPreset {
  return (
    MATERIAL_PRESETS.find((material) => material.id === id) ??
    MATERIAL_PRESETS[0]
  );
}
```

<Note>
  `metalness` and `roughness` map directly to the identically named properties on `THREE.MeshStandardMaterial`. The preset `color` is used as the base color for the material, but it can be overridden on a per-shape basis via `ShapeSettings.color`.
</Note>
